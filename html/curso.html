Identifica o servidor onde o recurso está localizado. Pode ser um nome de domínio, como "www.exemplo.com", ou um endereço IP. : O número de porta. Indica o número de porta TCP usado para acessar o serviço no servidor. Para protocolos comuns, como HTTP, o padrão é 80 para HTTP e 443 para HTTPS. : Este é o caminho (path) do recurso no servidor. Especifica o local no servidor onde o recurso está localizado. Por exemplo, em um site, o caminho pode ser "/pasta/pagina.html". : A query (consulta) ou parâmetros da URL. Esta parte opcional da URL é usada para enviar dados para o servidor. Os parâmetros são geralmente na forma de chave/valor e são separados por "&". Por exemplo, "?id=123&nome=exemplo". : O fragmento (fragment). Também é uma parte opcional da URL. É usado para identificar uma parte específica do recurso referenciado na URL. Por exemplo, em uma página da Web longa, o fragmento pode ser usado para rolar diretamente para uma seção específica da página. O fragmento é precedido por "#".
